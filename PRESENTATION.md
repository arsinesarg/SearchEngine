# Поисковый движок для сайта RAU

---

## Проблема

Стандартный поиск на сайте университета неудобен — нельзя быстро найти конкретного человека, кафедру или новость по ключевому слову.

---

## Решение

Собственный поисковый движок: краулер обходит сайт, строит индекс, сервер отвечает на запросы через веб-интерфейс в реальном времени.

---

## Архитектура

```
crawler  ──→  index.txt / urls.txt  ──→  server  ←──  браузер
```

| Компонент | Роль |
|-----------|------|
| `crawler` | Обходит сайт по BFS, извлекает текст, сохраняет индекс на диск |
| `server`  | Загружает индекс, отдаёт веб-UI и отвечает на поисковые запросы по HTTP |
| `client`  | CLI-альтернатива для поиска из терминала |

---

## Как работает индекс

Инвертированный индекс — для каждого слова хранится список страниц где оно встречается:

```
ректор      →  /about, /contacts
математика  →  /faculty/math
корпус      →  /news/new-building
```

Поиск по нескольким словам использует **AND логику** — возвращаются только страницы где присутствуют все слова запроса:

```
"новый корпус"  →  пересечение страниц с "новый" и "корпус"
```

## Технические решения

### UTF-8 и Кириллица

Стандартный `std::regex` работает побайтово и ломается на многобайтовых символах. Написан ручной байтовый парсер который корректно обрабатывает русский текст:

```
Привет  →  [ П=0xD0 0x9F ] [ р=0xD1 0x80 ] [ и=0xD0 0xB8 ] ...
МЕРИ    →  мери   (uppercase Cyrillic → lowercase побайтно)
```

### Очистка HTML

Перед индексацией из страницы удаляются невидимые блоки целиком — иначе слова из JavaScript и CSS попадают в индекс и дают ложные результаты:

```
<script>  ... </script>   ← удаляется полностью
<style>   ... </style>    ← удаляется полностью
<head>    ... </head>     ← удаляется полностью
```

### Веб-интерфейс

Сервер сам отдаёт HTML страницу — без фреймворков, встроена как raw string literal в C++:

- Поиск с мгновенным результатом
- Чипы для каждого слова запроса — кликни × чтобы убрать слово
- Пагинация по 5 результатов на страницу
- Розовый градиентный фон, тёмный текст

### Многопоточный сервер

Запросы обрабатываются параллельно через thread pool на 4 потока. При завершении потоки корректно останавливаются через stop-флаг и `join`.

## Стек

- **C++17** — весь проект, без фреймворков
- **libcurl** — HTTP запросы в краулере
- **POSIX sockets** — TCP сервер
- Внешних зависимостей кроме libcurl нет

---

## Тестирование

52 автоматических теста покрывают всю основную логику:

| Группа | Что проверяется |
|--------|----------------|
| `toLower` | ASCII и Кириллица, граничные случаи |
| `tokenize` | Разбивка, фильтрация, смешанный текст, регистр |
| `findCI` | Поиск без учёта регистра |
| `stripBlocks` | Удаление script/style блоков, регистр тегов |
| `extractText` | Полный пайплайн очистки HTML, entities |
| `search` | AND-логика, нет результатов, регистр запроса |

```
$ make test
52 passed, 0 failed
```

```bash
# Шаг 1 — проиндексировать сайт
./crawler

[2] Indexed: https://rau.am/about
...
Done. Pages: 30  Unique words: 4821

# Шаг 2 — запустить сервер
./server
Index loaded: 4821 terms, 30 pages
Server listening on http://localhost:8080
```

Открыть в браузере: **http://localhost:8080**
